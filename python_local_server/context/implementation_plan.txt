[KEY FINDINGS]
==============
1. Multiple alarms per category+vendor+tech:
   - SITE_DOWN + 4G + Huawei -> SQL returns 2 alarms:
     * Enodeb Out of service
     * Ne is disconnected
   -> 2 API calls (one per alarm) — CURRENT CODE IS CORRECT

2. Delta / Upper -> Pipe-separated location (1 API call):
   delta         -> location: "East Delta|West Delta"
   upper         -> location: "North Upper|South Upper"
   upper egypt   -> location: "North Upper|South Upper"
   Service script splits by | => creates IN condition for both

3. Date defaults to TODAY if not specified:
   - No time mentioned   -> today 00:00:00 to 23:59:59
   - "last 2 days"       -> now minus 2 days
   - "last week"         -> now minus 7 days
   - "yesterday"         -> yesterday full day
